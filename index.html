🧩 圧縮プロンプト生成
ここで作るのは単なる要約ではなく、次の執筆で破綻を防ぐための状態圧縮プロンプトです。 前回圧縮メモを更新元に1件置き、今回増えた本文や最新アウトラインだけを圧縮対象にすると、トークン節約しやすくなります。 章設計用・直前接続用も、ここから目的別に作る運用が基本です。
初回作成向けです。選択した資料から1本の新しい圧縮メモを作る前提でプロンプトを出します。
既存圧縮メモを全面再読させる代わりに、前回圧縮メモをここで1件だけ更新元として渡し、今回増えた本文や最新アウトラインだけを圧縮対象にすると効率的です。
本文 / アウトライン / 補助メモを複数選択できます。差分更新時は、更新元メモを上で1件選び、ここでは新たに増えた本文や最新アウトラインだけに絞るのがおすすめです。
🧭 圧縮プロンプトの使い方
執筆保管庫の中心機能は 圧縮プロンプト です。章設計メモと直前接続メモは独立した手書きメモではなく、 圧縮プロンプトで作った圧縮結果の保存先として使います。
① 全体整合性を更新する
使うもの：前回の全体整合性メモ / 最新本文 / 必要なら最新アウトライン
圧縮目的：全体整合性用
圧縮方式：既存圧縮メモを更新
効果：設定・因果・キャラ状態の破綻防止
② 次の章を設計する
使うもの：全体整合性メモ / 対象章アウトライン / 必要な補助メモ
圧縮目的：章設計用
保存先：章設計メモ
効果：この章で進める感情・仕込み・回収・禁止事項を整理
③ 本文を書く直前に接続を作る
使うもの：直近本文 / 全体整合性メモ / 必要なら章設計メモ
圧縮目的：直前接続用
保存先：直前接続メモ
効果：感情温度・距離感・物理状況・継続中の因果を接続
おすすめ運用： 毎回本文全量をAIへ渡すのではなく、まず全体整合性メモを差分更新し、その後に章設計用・直前接続用へ分岐します。 こうするとトークン節約と破綻防止を両立しやすくなります。
入口の考え方： 「🧩 章設計用を作る」「🔗 直前接続用を作る」は圧縮プロンプトの目的別ショートカットです。 圧縮プロンプトで整えた結果を、それぞれ章設計メモ / 直前接続メモとして保存してください。
